# /screen-spec — 화면정의서(스토리보드)

`spec-writer` 에이전트를 사용해 **화면정의서**를 만듭니다.

## 절차
1. `templates/화면정의서.md` 구조를 확인한다.
2. 와이어프레임·유저플로우·기능명세를 참조해 화면 ID를 일치시킨다.
3. `spec-writer`에게 위임: 화면별 진입경로, 영역별 구성요소, 요소 동작·유효성, 분기·예외를 디스크립션 표로.
4. 완료 후 `/doc-review` → `/drive-sync` 안내.

인자가 없으면 대상 화면과 와이어프레임 위치를 먼저 묻는다.
