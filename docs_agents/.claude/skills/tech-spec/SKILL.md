# /tech-spec — 기술명세서

`spec-writer` 에이전트를 사용해 **기술명세서**를 만듭니다. 요구사항정의서의 각 REQ를
개발자가 그대로 구현 가능한 **기술구현방식·기술비기능·처리흐름·예외**로 전개합니다.

## 절차
1. **매체 사전확정 게이트(필수)**: 착수 전 산출 매체를 1줄 확인한다. 상위 **요구사항정의서가 Sheets면
   기술명세서도 Sheets로 승계**한다(§CLAUDE.md 5). 표 중심이라 Sheets가 기본.
2. `templates/기술명세서.md` 구조(10컬럼: ID·모듈·기능명·사용자가치·기술구현방식·기술비기능·처리흐름·예외·Phase·우선순위)를 확인한다.
3. 요구사항정의서(`02_요구사항`)를 참조해 각 REQ를 기술 명세로 전개한다. REQ ID와 1:1 추적성을 유지한다.
4. `spec-writer`에게 위임: 구현 방식·핵심 알고리즘·데이터 흐름, **측정 가능한 비기능 임계치**, 처리흐름, 예외 케이스.
   근거 없는 값은 `(확인 필요)`로 막는다.
5. 완료 후 `/doc-review` → `/drive-sync` 안내.
   - Sheets 산출 시 **서식은 XLSX 빌드→import**(`docs/DOC-FORMATTING.md §3-A`). Phase·우선순위 드롭다운, P0 조건부서식.

인자가 없으면 대상 시스템·요구사항 문서 위치·산출 매체를 먼저 묻는다.
