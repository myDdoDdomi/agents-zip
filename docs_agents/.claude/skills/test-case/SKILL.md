# /test-case — 테스트케이스

`qa-engineer` 에이전트를 사용해 **테스트케이스**를 만듭니다.

## 절차
1. `templates/테스트케이스.md` 구조를 확인한다.
2. 요구사항(인수기준)·기능명세·API명세를 참조한다.
3. `qa-engineer`에게 위임: 동등분할/경계값으로 정상·예외·경계 케이스 도출, 요구사항 커버리지 표.
4. 완료 후 `/doc-review` → `/drive-sync` 안내.

인자가 없으면 대상 기능과 요구사항 문서 위치를 먼저 묻는다.
