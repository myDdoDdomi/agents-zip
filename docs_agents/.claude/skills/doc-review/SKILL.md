# /doc-review — 문서 검수·첨삭

`doc-reviewer` 에이전트를 사용해 **모든 문서의 검수·첨삭**을 수행합니다.

## 절차
1. 검수 대상을 확보한다:
   - 사용자가 붙여넣은 텍스트, 또는
   - Drive 문서(있으면 Workspace MCP로 읽기), 또는 직전 단계에서 생성한 초안.
2. 문서 종류를 식별하고 해당 `templates/`·`docs/STYLE-GUIDE.md`를 기준으로 삼는다.
3. `doc-reviewer`에게 위임: 체크리스트 기반 점검 → 심각도(🔴/🟡/🟢) 분류 → `위치|문제|근거|수정 제안(전→후)` 표 → 종합 의견·판정.
4. 사용자가 수정을 승인하면 **지시한 대로만** 반영하고 변경 요약을 보고한다.
5. 반영이 끝나면 `/drive-sync`로 Drive 반영을 안내한다.

## 주의
- 임의로 의미를 바꾸지 않는다. 제안은 제시하되 채택은 사용자가 결정한다.
- 모든 지적에는 근거(스타일가이드/템플릿/품질기준)를 단다.
