# /ia — 정보구조도(IA)

`ux-architect` 에이전트를 사용해 **정보구조도(IA)** 를 만듭니다.

## 절차
1. `templates/정보구조도-IA.md` 구조를 확인한다.
2. 기능명세/요구사항을 참조해 화면·콘텐츠를 계층(Depth 1·2·3)으로 분류한다.
3. **Depth 분류 시 IA Depth 규칙을 적용한다**(템플릿 §1-1): 1Depth=기능 모듈명(화면명 금지),
   2Depth=Step 화면(순차여도 상하관계 아니라 병렬), 3Depth=특정 화면 내부에서만 열리는 하위 화면.
   **모든 화면에 Type(Page/Popup)을 지정**한다(Popup은 호출 화면보다 한 단계 아래 Depth).
4. `ux-architect`에게 위임: 일관된 화면 ID 부여, Mermaid `graph` 또는 트리로 시각화.
5. **화면별 컴포넌트 인벤토리 작성**: 각 화면에 보이는 **모든 컴포넌트를 빠짐없이** 표로 남긴다
   (컴포넌트명·유형·역할/내용·상태·인터랙션/동작·데이터 바인딩·제약). 헤더/네비/버튼/입력필드/라벨/
   아이콘/이미지/리스트/카드/모달/토스트/빈상태/로딩/에러까지. 목적은 **이해관계자 간 누락·해석 차이를 좁히는 것** —
   템플릿의 누락 점검 체크리스트로 확인한다.
6. **완료 전 IA Depth 검증 체크리스트 5항목 통과를 확인한다**(템플릿 §1-1):
   ① 1Depth에 화면명 없음 ② 순차 Step을 Depth 증가로 처리한 곳 없음 ③ 전 화면에 Type 지정
   ④ Popup이 호출 화면과 같은 Depth 아님 ⑤ 화면 카운트에서 1Depth 모듈명 제외.
7. 완료 후 `/doc-review` → `/drive-sync` 안내. (후속: `/menu-tree`, `/user-flow`)

인자가 없으면 대상 서비스 범위를 먼저 묻는다.
