# /wireframe — 와이어프레임

`ux-architect` 에이전트를 사용해 **텍스트 와이어프레임**을 만듭니다.

## 절차
1. `templates/와이어프레임.md` 구조를 확인한다.
2. 화면 ID(IA/메뉴트리)와 유저플로우를 참조한다.
3. `ux-architect`에게 위임: 화면별 ASCII/박스 레이아웃 + **컴포넌트 인벤토리** 표.
4. **컴포넌트 인벤토리는 화면에 보이는 모든 요소를 빠짐없이** 표로 남긴다
   (컴포넌트명·유형·역할/내용·상태·인터랙션/동작·데이터 바인딩·제약). 헤더/네비/버튼/입력필드/라벨/
   아이콘/이미지/리스트/카드/모달/토스트/빈상태/로딩/에러까지. 목적은 **이해관계자 간 누락·해석 차이를 좁히는 것** —
   템플릿의 누락 점검 체크리스트로 확인한다.
5. 완료 후 `/doc-review` → `/drive-sync` 안내. (후속: `/screen-spec`)

인자가 없으면 대상 화면을 먼저 묻는다.
