# /drive-sync — Google Drive 반영 (Docs + Sheets, 서식 포함)

완성/수정된 문서를 팀 공유 Drive에 **가독성·가시성 높은 네이티브 서식**으로 안전하게 반영합니다.

## 사전 확인
1. `config/drive-config.md`에서 **대상 폴더 ID**와 **파일명 규칙**을 확인한다.
2. Google Workspace MCP가 연결돼 있는지 확인한다(도구에 docs/sheets/drive 검색).
   - **연결 안 됨**: Drive 쓰기를 시도하지 말고 마크다운 산출물을 제공, `docs/GOOGLE-DRIVE-MCP.md`로 연결 안내(폴백 모드).

## 형식 선택 (Docs vs Sheets)
- **Sheets로**: 요구사항정의서, 기능명세서, API명세서, DB설계서, 테스트케이스, 화면정의서, 문서관리대장
- **Docs로**: 기획서, 회의록, 릴리즈노트, 검수/분석 리포트, IA, 유저플로우, 시퀀스다이어그램, 와이어프레임
- 기존 문서가 있으면 그 형식을 따른다.

## 절차
1. **중복 확인**: 대상 폴더에서 같은 이름의 문서를 검색한다.
   - 있으면 → 기존 문서를 읽어 내용을 파악하고, 바꿀 부분을 사용자에게 보여준 뒤 **이어쓰기/섹션·행 교체**를 우선한다.
     전체 덮어쓰기는 손실 위험을 알리고 승인받은 뒤에만.
   - 없으면 → `config/drive-config.md` 규칙에 맞춰 새 문서를 만든다. **본문을 plain text로 통째로 넣지 않는다.**
2. **서식은 `doc-formatter` 에이전트에 위임한다.** (직접 plain text를 밀어넣지 말 것)
   - `doc-formatter`가 `docs/DOC-FORMATTING.md` 표준대로 커버 헤더·제목 스타일·네이티브 표·브랜드 색(#1F3A5F)을 입힌다.
   - **Sheets로 작성/수정**할 때는 `docs/DOC-FORMATTING.md` **§3-1(작성 절차·도구·A1 범위·문서종류별 컬럼·수정 안전절차)** 를 따른다 — 헤더 1행은 컬럼=템플릿 일치, 데이터는 2행부터, 수정 시 헤더 보존. 헤더 고정·색·줄바꿈·필터까지 적용한다.
3. 반영 후 **문서 URL**을 사용자에게 보고한다. (새 Doc은 보통 내 드라이브 루트에 생기므로 대상 폴더로 이동까지 확인)

## 중복 생성 방지
- 같은 문서를 만들 때는 **한 번만 생성**한다. 병렬로 여러 번 생성하지 말 것.
- 생성 후 같은 이름이 여러 개면 사용자에게 알리고 정리 여부를 묻는다(삭제는 승인 후).

## 안전 규칙
- 덮어쓰기·삭제 전에는 항상 현재 내용을 읽고, 손실 위험이 있으면 먼저 알린다.
- 인증/권한 오류 시 원인과 해결(재인증, 폴더 공유 권한)을 안내한다.
- 비밀값(토큰/키)은 출력하지 않는다.
