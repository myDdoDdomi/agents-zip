# /office-hours — 요구 인터뷰 (착수 전 인테이크)

`intake-interviewer` 에이전트를 사용해 **문서를 쓰기 전에** 요구를 명확히 합니다.
불명확한 입력으로 시작해 동일 산출물을 반복 재생성하는 낭비를 막는 첫 게이트입니다.

## 절차
1. 대상 주제와 (있으면) 상위 문서 위치를 확인한다. 인자가 없으면 무엇을 만들려는지 먼저 묻는다.
2. `intake-interviewer`에게 위임: **6대 포싱 질문**(누구를·진짜 문제·성공 측정·범위 밖·위험한 가정·매체/기한/제약)을 한 번에 3~5개씩 핵심부터.
3. 답을 받아 **인테이크 브리프**(확정 사실 / 미해결 질문(차단요인) / 가정+검증책임자 / 권장 산출 매체 / 다음 문서)를 산출한다.
4. 차단요인(미해결 질문)이 남아 있으면 그것부터 해소하고, 없으면 다음 문서로 넘어간다.

완료 후 안내: "다음: `/planning-doc`(기획) 또는 `/requirements-doc`(요구사항)". 표 중심이면 §5 매체 게이트(Sheets/Docs) 확인.
