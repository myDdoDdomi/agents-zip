# Drive 매핑 설정 (팀 공유)

> 팀 전체가 **같은 Google Drive 계정/폴더**를 바라보도록, 여기에 공유 Drive 위치를
> 한 번 정의해 두면 모든 에이전트가 이 위치에 문서를 저장/수정합니다.
> 이 파일은 레포에 커밋되어 팀원 전체가 공유합니다. (개인 인증 정보는 여기에 넣지 마세요.)

## 공유 루트 폴더

| 항목 | 값 |
|---|---|
| 공유 드라이브/폴더 이름 | `(예: 우리팀-문서)` |
| 공유 폴더 ID | `(TBD - Drive 폴더 URL의 .../folders/뒤 ID를 붙여넣기)` |
| 소유 계정(공유 계정) | `(예: team-docs@회사.com)` |

> 폴더 ID 찾는 법: Drive에서 폴더 열기 → 주소창
> `https://drive.google.com/drive/folders/`**`이부분이ID`**

## 사용 규칙 (에이전트가 따름)

1. 문서를 Drive에 만들 때 위 표의 **하위 폴더 ID**를 우선 사용합니다.
   ID가 `(TBD)`면 사용자에게 폴더 ID를 묻거나, 루트 폴더에 만들고 알립니다.
2. 같은 이름의 문서가 이미 있으면 **새로 만들지 말고** 기존 문서를 열어
   비교 후 업데이트합니다. (덮어쓰기 전 §CLAUDE.md 5 규칙 준수)
3. 파일명 규칙을 지켜 검색/정렬이 쉽게 합니다.
