# 작업 워크플로우

모든 문서는 **4단계 루프**를 따릅니다.

```
① 초안(Draft) → ② 검수(Review) → ③ 수정(Revise) → ④ 반영(Sync)
```

## ① 초안 (Draft)
- 문서 종류 슬래시 스킬을 호출 → 담당 에이전트가 템플릿에 맞춰 작성.
- 정보가 부족하면 에이전트가 질문합니다. 답하면 초안 품질이 올라갑니다.
- 상위 문서(기획→요구사항→명세…)가 Drive에 있으면 먼저 참조해 일관성을 맞춥니다.

```
/requirements-doc 출석체크 요구사항 정의서 — 기획서는 Drive 01_기획 폴더에 있어
```

## ② 검수 (Review)
- `/doc-review` 로 `doc-reviewer`의 첨삭을 받습니다.
- 결과는 `위치 | 문제 | 근거 | 수정 제안(전→후)` 표 + 심각도(🔴/🟡/🟢) + 판정으로 옵니다.

```
/doc-review 방금 만든 요구사항 정의서 검수해줘
```

## ③ 수정 (Revise)
- 검수 의견 중 **반영할 것을 골라 지시**하면, 그 지시대로만 수정합니다.
- 에이전트가 임의로 의미를 바꾸지 않습니다("입력하는 대로" 반영 원칙).

```
🔴 1, 2번은 제안대로 반영하고, 🟡 3번은 이렇게 바꿔줘: ...
```

## ④ 반영 (Sync)
- `/drive-sync` 로 Google Drive 문서에 저장/업데이트합니다.
- 같은 이름 문서가 있으면 **이어쓰기/섹션 교체**를 우선하고, 전체 덮어쓰기는 확인 후 진행.
- 완료되면 문서 URL을 받습니다.

```
/drive-sync 06_API 폴더에 올려줘
```

---

## 문서 의존성 (작성 순서 권장)
```
기획서 → 요구사항정의서 → 기능명세서 → 화면정의서
                          ├→ IA → 메뉴트리 → 유저플로우 → 와이어프레임
                          ├→ API명세서 → 시퀀스다이어그램
                          └→ DB설계서
                                  → 테스트케이스 (요구사항/명세 기반)
```
앞 문서가 있으면 뒤 문서 작성 시 자동으로 참조해 ID·용어를 일치시킵니다.

## 협업 팁
- **버전**: 문서 상단 표의 `문서 버전`을 수정 때마다 올립니다(v0.1 → v0.2 …).
- **충돌 방지**: 같은 문서를 동시에 편집하지 말고, Drive 댓글/상태로 소통합니다.
- **추적성**: 요구사항 ID ↔ 기능 ID ↔ 화면/API/TC ID를 끊지 마세요. 검수에서 점검합니다.
