# 한국어 문서 스타일 가이드

모든 에이전트와 팀원이 따르는 **문서 작성 절대 기준**입니다. 검수(`/doc-review`)는
이 가이드 위반을 지적합니다.

## 1. 언어·어조
- **한국어** 사용. 문서 본문은 간결한 **평서체(개조식/서술 혼용)**, 안내문은 정중한 존댓말.
- 고유 기술 용어·제품명·식별자는 원문(영어) 유지. 최초 등장 시 한글 설명 병기.
  - 예: "인증 토큰(access token)을 발급한다."

## 2. 명확성
- **모호어 금지**: "적절히", "등", "필요시", "추후", "기타" → 구체적 수치/조건으로.
- **한 문장 한 메시지**. 긴 문장은 나눈다.
- **측정 가능하게**: 요구사항/완료조건/기대결과는 검증 가능한 표현으로.
  - ✗ "빠르게 응답한다" → ✓ "p95 응답시간 500ms 이하"

## 3. 용어·표기 일관성
- 같은 개념은 **같은 단어**로. (예: "회원" vs "사용자" 혼용 금지 — 하나로 통일)
- 약어는 **최초 1회 풀어쓰기**: "단일 페이지 애플리케이션(SPA)".
- 식별자 체계 통일: 요구사항 `FR-001`/`NFR-001`, 기능 `FN-001`, 화면 `SCR-영역-001`,
  API `API-001`, 테스트 `TC-001`.

## 4. 숫자·날짜·단위
- 날짜: `YYYY-MM-DD`. 시간: 24시간제 `HH:MM`.
- 숫자: 천 단위 콤마(`1,000`). 단위는 값에 붙여 표기(`500ms`, `20개`, `3초`).
- 범위: `8~64자`처럼 물결(~) 사용.

## 5. 구조·서식
- 문서 상단에 **메타 표**(버전/작성자/작성일/관련 문서)를 둔다.
- 제목 계층은 `#`(문서 제목) → `##`(대단원) → `###`(소단원) 순서로만.
- 목록·비교·매핑은 **표**로, 흐름·구조는 **Mermaid**로 표현.
- 미정 항목은 `(TBD - 작성 필요)`, 가정은 `(가정: …)`, 확인 필요는 `(확인 필요: …)`.

## 6. 다이어그램 규칙
- 기본은 **Mermaid**(GitHub·Drive 어디서나 재현 가능). 종류:
  - 흐름: `flowchart`, 시퀀스: `sequenceDiagram`, ERD: `erDiagram`, 상태: `stateDiagram-v2`
- 다이어그램 아래에는 **글로 된 설명**을 덧붙인다(다이어그램만으로 끝내지 않기).

## 7. 금지/주의
- 사실이 아닌 내용을 단정하지 않는다. 근거 없는 수치 금지.
- 비밀값(토큰/키/개인정보)을 문서에 넣지 않는다.
- 템플릿 섹션을 임의 삭제하지 않는다(불필요하면 `(해당 없음)` 표기).

## 8. 체크(셀프)
- [ ] 모호어 없는가
- [ ] 용어/ID 일관적인가
- [ ] 측정·검증 가능한가
- [ ] 표/다이어그램이 적절히 쓰였는가
- [ ] 메타 표·버전이 갱신됐는가
