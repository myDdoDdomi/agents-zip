# 예시 — 간트차트 + 칸반보드 (한 구글 스프레드시트)

> 가상의 프로젝트 **"모바일 출석체크 앱 v1.0"** (기준일 2026-06-01, 월요일)로 만든 예시입니다.
> `project-scheduler` 에이전트가 `/project-plan`으로 만들어 내는 산출물의 형태를 보여줍니다.
> **하나의 스프레드시트 = 4개 시트(탭)**, `Tasks`가 입력원이고 `Gantt`·`Kanban`·`This Week`는 이를 읽어 표시합니다(단방향).

이 폴더의 `*.csv`를 구글 시트로 가져오면(아래 §올리는 법) 그대로 시트가 됩니다.

> T7은 신뢰도 **낮음** → 추정 전 **스파이크 T0**를 선행으로 두고, `(확인 필요: 기간 — 한프론트가 6/12까지 확정)`.

- 상태 값: `대기 / 진행중 / 검수중 / 완료 / 지연 / 차단` — 이 값이 간트 막대 색과 칸반 분류를 결정합니다.
  - **`차단`**(예: T8 — 선행/외부요인에 막힘, 주황)은 `지연`(종료일 경과, 빨강)과 구분하고, 비고에 **차단사유·차단원인 Task ID**를 적습니다.
  - **`검수중`은 코드리뷰+QA를 포함**하는 단계입니다(QA는 QA_agents 핸드오프 가능 — 예: T9).
- **개발자 관점(경량) 반영:** 개발 task(T6)에 **리뷰·재작업 버퍼(~15~20%)** 포함, 추정 **신뢰도(높음/중간/낮음)**,
  완료조건(DoD), 불확실 task(T7)는 `(확인 필요: 기간, 책임자·기한)`과 신뢰도=낮음 → **스파이크(T0) 선행** 권고.

## 올리는 법 (구글 드라이브)

### 방법 A — CSV를 직접 가져오기 (지금 바로)
1. [Google Sheets](https://sheets.new)에서 새 스프레드시트 생성 → 이름: `모바일 출석체크 앱 v1.0 — 일정`.
2. 시트(탭)를 `Tasks` / `Gantt` / `Kanban` / `This Week` 4개로 만든다.
3. 각 탭에서 **파일 → 가져오기 → 업로드**로 같은 이름의 `.csv`를 넣는다(가져오기 위치: "현재 시트에 바꾸기").
4. 가독성 마무리: 헤더 1행 **틀고정**, 헤더 배경 `#1F3A5F`+흰 굵은 글씨, 긴 열 **줄바꿈+행높이 자동**, 이번 주(W1) 열 배경 강조, 상태 열 색 구분. (= `docs/DOC-FORMATTING.md` §3-0)

### 방법 B — 에이전트가 드라이브에 직접 생성 (자동)
이 세션이 아니라 **`docs_agents` 폴더를 Claude Code로 직접 열어야** 합니다(거기에 Google Workspace MCP가 설정돼 있음).
1. `docs_agents/`를 Claude Code로 연다 → `/mcp`로 `google-workspace` 서버 **승인** → 첫 실행 시 **구글 OAuth 1회**(Calendar·Sheets 권한 포함).
   - (설치/환경변수: `docs/GOOGLE-DRIVE-MCP.md`)
2. 프롬프트: `/project-plan 모바일 출석체크 앱 v1.0` 또는 "이 예시로 간트+칸반 스프레드시트 만들어서 드라이브에 올려줘".
3. `project-scheduler`가 위 4개 시트를 만들고 서식·수식까지 적용해 **공유 드라이브 폴더**에 저장합니다.

> ⚠️ **단방향**: 상태 변경은 항상 `Tasks` 시트에서. 칸반에서 카드를 끌어 옮겨도 `Tasks`로 자동 반영되지는 않습니다(시트 수식의 한계).
