> 구현 가능한 **기술 구현 방식·비기능·처리흐름·예외**로 전개한다. 표 중심이라 **Sheets로 산출**하는
> 것이 기본이며, 상위 요구사항정의서가 Sheets면 **반드시 Sheets로 승계**한다(매체 사전확정 게이트).

### 2-1. 컬럼 작성 가이드

| 컬럼 | 무엇을 적나 | 셀 표기 규칙 |
|---|---|---|
| **ID** | 요구사항 ID(상위 요구사항정의서의 REQ ID와 1:1 연결) | `REQ-001` 형식. 추적성 유지 |
| **모듈** | 소속 모듈/도메인 | 짧은 명사(인증, 결제, 알림 …) |
| **기능명** | 기능의 이름 | 동사+목적어(로그인, 토큰 갱신 …) |
| **사용자가치** | 이 기능이 주는 가치(왜) | 한 문장. 모호어("적절히") 금지 |
| **기술구현방식** | 어떻게 구현하나(핵심) | 접근·알고리즘·라이브러리·데이터 흐름·캐시 키·트랜잭션 경계 등 구체적으로. 검증 가능하게 |
| **기술비기능** | 성능·보안·확장·관측 요건 | **측정 가능한 임계치**(예: p95 < 300ms). 임계치 불명이면 `(확인 필요)` |
| **처리흐름** | 정상 흐름 단계 | `1. … 2. …` 번호. 분기·외부호출 표시 |
| **예외** | 실패·경계·에러 | `조건 → 처리·메시지`. 빈칸 금지(없으면 "없음" 명시) |
| **Phase** | 출시 단계 | `MVP / Phase2 / Phase3` (드롭다운 표준화) |
| **우선순위** | 구현 우선순위 | `P0 / P1 / P2 / P3` (드롭다운 + P0=빨강 조건부서식) |

### 2-2. 셀 표기 규칙 (Sheets 산출 시)
- **Sheets 서식은 `openpyxl` XLSX 빌드 → import**(`docs/DOC-FORMATTING.md §3-A`). 직접 셀서식 API 없음.
- **Phase·우선순위는 드롭다운**(DataValidation)으로 입력 표준화(오타 방지).
- **우선순위 P0은 조건부서식**(FormulaRule)으로 빨강 강조.
- 긴 텍스트 열(기술구현방식·기술비기능·처리흐름·예외)은 **줄바꿈(wrap) + 상단 정렬**, 헤더 1행 **틀고정**.
- 근거 없는 값은 지어내지 말고 `(확인 필요: …)`로 표기(§CLAUDE.md 운영 원칙 3).
