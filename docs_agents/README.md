# docs-agents 📝

> 우리 팀의 **AI 문서화 팀**입니다. 한국어로 시키기만 하면 기획·개발 문서를 표준 형식에 맞춰
> 대신 써주고, 검토받게 해주고, **Google Drive에 정리**까지 해줍니다. **개발 지식이 없어도 쓸 수 있어요.**

`garrytan/gstack`(Claude Code를 가상 엔지니어링 팀으로 만드는 스킬 툴킷)의 검증된 아이디어
3가지를 문서팀 맥락으로 이식했습니다: **① 요구 인터뷰**(`/office-hours` — 착수 전 포싱 질문으로
요구·갭·가정·성공기준 확정), **② Diátaxis 문서 분류**(튜토리얼/하우투/레퍼런스/설명 4분면으로
문서 성격을 먼저 정함), **③ 다관점 심화 검수**(`/deep-review` — 중요 문서를 5개 렌즈로 병렬 검수 후
1개 판정으로 종합).

## 🟢 Claude Code 처음이세요? — 설치 1분

터미널(명령을 입력하는 검은 창)을 열고 **자기 컴퓨터에 맞는 아래 한 줄을 복사 → 붙여넣기 → Enter** 하면 끝납니다.

**🍎 macOS** — `Terminal(터미널)` 앱 실행 후:

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**🪟 Windows** — 시작 메뉴에서 `PowerShell` 검색해 실행 후:

```powershell
irm https://claude.ai/install.ps1 | iex
```

설치가 끝나면 같은 창에 **`claude`** 라고 입력하고 Enter 하세요. 처음 한 번만 **브라우저가 열려 로그인**(Claude Pro/Max/Team 또는 Console 계정)하면 되고, 그다음부터는 `claude`만 치면 바로 시작됩니다.

> 막히면 공식 설치 가이드: <https://code.claude.com/docs/en/quickstart> · Windows는 `Git for Windows`가 깔려 있으면 더 매끄럽습니다.

## 🚀 이 팀 쓰는 법 (3단계)

1. **이 폴더를 받아 `claude`로 엽니다.** 이 `docs_agents` 폴더를 컴퓨터로 받은 뒤, 그 폴더 안에서
   터미널에 `claude`라고 입력해 엽니다. (그러면 문서팀 AI들이 자동으로 준비됩니다.)
2. **Google Drive를 1회 연결합니다.** 만든 문서가 Google Drive에 저장되도록, 처음 한 번만 본인 구글 계정을
   연결합니다. → [연결 가이드](docs/GOOGLE-DRIVE-MCP.md)
3. **입력창에 한국어로 시킵니다.** 예: `출석 체크 기능 기획서 만들어줘`
   → AI가 초안 작성 → "검수해줘" → "고쳐줘" → "드라이브에 올려줘" 순으로 자연스럽게 이어집니다.

> 🟢 **완전 처음이라면** 단계별로 친절하게 설명한 [비개발자용 사용 설명서](docs/GETTING-STARTED.md)부터 보세요.

## 🤖 "에이전트"와 "스킬"이 뭐예요? (1분 개념)

어렵지 않아요. 딱 두 가지만 알면 됩니다.

- **에이전트(agents) = AI 팀원.** 각자 한 분야 전문가예요. 당신이 한국어로 시키면 **그룹장이 알맞은 팀원에게 알아서 맡깁니다.** → *팀원 이름을 외울 필요가 없어요.*
- **스킬(skills) = `/`로 시작하는 단축 명령.** 그냥 말로 시켜도 되고(예: "기획서 만들어줘"), 콕 집어 부를 수도 있어요(예: `/planning-doc`). 입력창에 **`/`만 쳐도 쓸 수 있는 명령 목록**이 뜹니다.

**결론: 외울 건 없습니다. 그냥 한국어로 하고 싶은 걸 말하세요.** 아래 표는 "이 팀이 뭘 해줄 수 있는지" 참고용이에요.

## 👥 이 팀의 팀원(agents)

| 팀원 | 무엇을 해줘요 (쉬운 말) |
|---|---|
| 요구 인터뷰어 (intake-interviewer) | 문서를 만들기 **전에**, 꼭 필요한 질문을 던져서 "누구를 위한 건지·진짜 문제·성공 기준·범위"를 먼저 정리해줘요. (헛수고 방지) |
| 리서처·분석가 (reference-researcher, insight-analyst) | 경쟁 제품·사례를 **출처와 함께** 모아주고, 거기서 쓸 만한 패턴·기회·아이디어를 뽑아줘요. |
| 기획·요구 정리팀 (doc-planner, requirements-analyst) | 기획서를 쓰고, 그 기획을 "무엇을 만들어야 하는지" 요구사항 목록으로 풀어줘요. |
| 명세 작성자 (spec-writer) | 기능명세서·기술명세서·화면정의서처럼 "정확히 어떻게 동작하는지"를 꼼꼼히 적어줘요. |
| 화면·구조 설계자 (ux-architect) | 정보구조도(IA)·메뉴 구조·유저 흐름·와이어프레임 등 화면 짜임새를 잡아줘요. |
| 설계 전문가들 (api-designer, data-modeler, diagram-author) | API 명세·DB 설계(ERD)·흐름도 같은 개발용 설계 문서를 만들어줘요. (개발자에게 넘기기 좋아요) |
| QA 엔지니어 (qa-engineer) | 만든 기능을 어떻게 테스트할지 테스트케이스를 짜줘요. |
| 테크니컬 라이터 (tech-writer) | 회의록·릴리즈노트(변경 안내문) 같은 글을 깔끔히 정리해줘요. |
| 일정 담당팀 (calendar-manager, project-scheduler) | 구글 캘린더 일정을 정리/등록하고, 간트차트·칸반보드로 프로젝트 일정 계획을 세워줘요. |
| 검수자 (doc-reviewer) | 완성한 문서를 꼼꼼히 검토·첨삭하고, 중요한 문서는 여러 관점으로 깊게 봐줘요. |
| 서식 전문가 (doc-formatter) | 검수 끝난 문서를 보기 좋게 서식 입혀 **Google Drive에 저장**해줘요. |
| 회고 담당 (feedback-reporter) | 작업을 돌아보고 "더 잘하는 법" 피드백을 정리해 본부에 전달해줘요. |

> 이름을 외울 필요는 없어요. 그냥 시키면 **그룹장이 알맞은 팀원에게 알아서 맡깁니다.**

## 🤝 다른 팀과 함께 쓰기 (멀티팀 조립)

> **에이전트만 복사해도 됩니다(자동 포트):** `대상/.claude/agents/<팀>/`에 에이전트만 복사하고 `/team-assemble`를
> 실행하면, 아래 수동 복사 단계(스킬·템플릿·참조 문서·`.mcp.json` 병합)는 스킬이 **본부 레포에서 자동으로
> 가져옵니다**(같은 머신에 본부 레포가 있을 때 — 경로는 1회만 물음, SKILL §2-1b). **환경변수 설정·브라우저 OAuth
> 1회 인증만 직접** 하면 됩니다. 본부 레포가 없는 환경에서는 아래 수동 절차를 그대로 따르세요.

이 문서팀을 **개발(dev)·QA 등 다른 팀과 한 프로젝트에서 같이** 쓰고 싶을 때 읽으세요.
(문서팀만 혼자 쓸 거면 — 이 `docs_agents` 폴더를 그냥 직접 열면 됩니다. 조립은 필요 없어요.)

문서팀은 **폴더를 직접 여는 팀(모델 A)** 이라, 다른 팀과 합칠 땐 **에이전트(팀원)만 옮기면 부족**해요.
슬래시 명령(스킬)·문서 템플릿·Drive 연결 설정까지 함께 옮겨야 제대로 동작합니다.

### 무엇을 복사하나 (4가지)

대상 프로젝트(여러 팀을 같이 쓸 폴더)로 아래 4가지를 옮깁니다.

1. **① 팀원(에이전트):** `docs_agents/.claude/agents/*` → `대상-프로젝트/.claude/agents/docs/`
2. **② 단축 명령(스킬):** `docs_agents/.claude/skills/*` → `대상-프로젝트/.claude/skills/` (슬래시 명령이 동작하려면 필수)
3. **③ 문서 템플릿:** `docs_agents/templates/*` → `대상-프로젝트/templates/` (에이전트가 참조하는 상대경로를 유지)
4. **④ Drive 연결 설정:** `docs_agents/.mcp.json` → `대상-프로젝트/.mcp.json`에 **합칩니다(병합)**.
   비밀값은 들어 있지 않고 `${ENV}` 형태로 두며, 같은 키가 이미 있으면 사용자에게 확인합니다.
   + 본부의 `team-assemble` 스킬 폴더도 대상 `.claude/skills/`로 함께 복사하세요.

### 실행

대상 프로젝트에서 **`/team-assemble`를 한 번** 실행하면 됩니다.
→ 팀원 이름이 겹쳐도 자동으로 정리되고(충돌 해결), 협업용 `CLAUDE.md`·`MEMORY.md`가 자동으로 만들어집니다.
팀을 더하거나 빼면 **다시 실행만** 하면 됩니다(여러 번 돌려도 안전).

### 알아둘 점

- 문서팀의 `CLAUDE.md`·`MEMORY.md`는 따라가지 않습니다 — 협업용 `CLAUDE.md`의 자동 블록과 공통 규칙으로 대체돼요.
- 회고 담당(`feedback-reporter`)은 조립 시 **하나로 통합**됩니다. `/feedback-agents` 한 번이면 합쳐진 **모든 팀**을 회고해 본부로 보고됩니다.

### 📁 Drive(MCP) 주의 — 1회 재인증

문서팀의 `.mcp.json`에는 **Google Drive(Workspace) 연결 설정**이 들어 있어요.
대상 프로젝트로 병합한 뒤, 본인 구글 계정으로 **브라우저에서 한 번 다시 연결**하면 됩니다(비밀값은 레포에 안 들어가요).
→ [Google Drive 연결 가이드](docs/GOOGLE-DRIVE-MCP.md)

> 더 자세히는 본부의 루트 `README.md`/`CLAUDE.md`(§2-C 조립)와 `/team-assemble` 스킬을 보세요.

아래는 팀을 직접 운영·확장하는 사람을 위한 내용입니다. 그냥 문서를 만들고 싶을 뿐이라면 안 봐도 됩니다.

## 폴더 구조

```
docs-agents/
├── CLAUDE.md                # 그룹장(팀 운영 기준)
├── .mcp.json                # 팀 공유 Google Drive 연결 설정 (비밀값 없음)
├── config/drive-config.md   # 공유 Drive 폴더 매핑 (팀 리드가 1회 채움)
├── .claude/
│   ├── agents/              # 에이전트 팀원(담당 AI)
│   └── skills/              # 문서종류별 빗금(/) 명령
├── templates/               # 한국어 문서 템플릿
├── examples/                # 범용 서식판 예시 (다른 팀 참고용)
├── feedback/                # 팀 작업 회고·피드백 보고서 (→ 본부) /feedback-agents
└── docs/                    # 사용 설명서 · 워크플로우 · 스타일 · 연결 가이드
```

## 팀에 기여하기 (새 문서 종류/에이전트 추가)

1. `templates/`에 템플릿 추가
2. `.claude/agents/`에 담당 에이전트 추가 (또는 기존 에이전트에 매핑)
3. `.claude/skills/<이름>/SKILL.md`에 빗금 명령 추가
4. `CLAUDE.md` §4 라우팅 표에 한 줄 추가
5. PR 생성

## 🧠 팀 메모리 (MEMORY.md) 사용법

이 팀은 두 층의 메모리를 쓴다.
- **CLAUDE.md** — 사람이 쓰는 안정적 규칙·라우팅(매 세션 전체 로드).
- **MEMORY.md** — 작업하며 쌓이는 학습(검증된 함정·우회법·환경 매핑·반복 교정). CLAUDE.md가 `@./MEMORY.md`로 자동 로드한다.

| 항목 | 내용 |
|---|---|
| **언제 적나** | 같은 실수 2회 · 환경/도구 특이사항 · 매 세션 다시 설명하게 되는 것 · 검수가 잡아준 것 |
| **어떻게** | 해당 섹션에 **한 줄**(가능하면 날짜·근거). 규칙으로 굳으면 CLAUDE.md로 승격 |
| **확인** | 세션 중 `/memory`로 로드된 메모리 파일 확인 |
| **용량** | 200줄 이하 유지. 초과 시 `memory/<주제>.md`로 분리(필요할 때만 읽힘) |
| **개인 메모리** | 개인·머신 한정 학습은 Claude Code 자동 메모리(`~/.claude/projects/.../memory/`)가 별도 관리(레포 비커밋). 팀 공유 학습은 MEMORY.md |

## 🧩 모델·프롬프트 기준 (Claude Opus 4.8)

이 팀은 현행 **Claude Opus 4.8**(별칭 `opus`, 1M 컨텍스트) 기준으로 운영한다. 라인업·근거·전체 표는 루트 `claude-opus-4.8-운영노트.md`.

- **모델 별칭 유지.** 에이전트 `model:`은 `opus`/`sonnet`/`haiku`/`inherit` 별칭으로 둔다(모델 ID 하드코딩 금지 — Claude Code가 현행 버전으로 해석). 4.8는 천장이 높아 **effort는 `high`에서 시작**, 필요 시 상향(반사적 `max` 금지).
- **담백한 지시.** 4.8는 지시를 문자 그대로 따른다 → "CRITICAL/반드시/무조건" 같은 과장 명령은 과잉발동을 부르니 "…할 때 X" 형태로.
- **위임·도구·메모리는 트리거 명시.** 4.8는 위임/검색/메모리를 보수적으로 쓴다 → `description`과 도구 설명에 **"~할 때 호출"** 을 박고, 그룹장 루프에 "작업 전 MEMORY 확인"을 둔다.
- **ask-rate·나레이션 보정.** 사소한 선택은 정하고 기록만(범위 변경·파괴적 작업·Drive 덮어쓰기만 확인). 도구 호출 사이 불필요한 나레이션은 줄인다.
