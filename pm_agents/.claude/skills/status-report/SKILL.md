# /status-report — 진행 현황·릴리스 보고서

`status-reporter` 에이전트가 진척·번다운·리스크·블로커를 이해관계자용 보고서로 요약합니다.

## 절차
1. **대상 확인:** 스프린트/에픽/버전·기간·수신 대상을 확인한다. 모호하면 1~2개 질문.
2. `status-reporter`에게 위임: JQL로 진척 조회 → 완료/진행/대기 비율·목표 달성도·번다운·리스크·블로커.
3. `templates/상태보고서.md` 구조로 정리. 각 수치에 **근거 JQL + 조회 시점**. "다음 행동"을 짧게 정리한다.
4. 핵심 요약을 보고한다.

## 주의
- **읽기 전용.** Jira에 쓰지 않는다(보고서는 `.md` 산출). 낙관 편향 없이 리스크를 사실대로 적는다.
- 대외용 가공·게시가 필요하면 docs_agents/marketing_agents로 핸드오프하거나 `(확인 필요)`로 안내한다.

인자(`$ARGUMENTS`)가 있으면 대상·기간으로 해석하고, 부족하면 먼저 묻는다.
