# /sprint-plan — 스프린트 계획

`pm-lead`가 현황을 확인하고 `sprint-planner`가 용량·목표·범위를 산출하는 계획 흐름입니다.

## 절차
1. **대상 확인:** 보드/프로젝트·기간·가용 인원을 확인한다. 모호하면 1~2개만 질문(`MEMORY.md` 매핑 확인).
2. **현황:** 필요 시 `jira-analyst`로 백로그·과거 속도를 먼저 읽는다(`pm-lead`가 요약).
3. **계획:** `sprint-planner`에게 위임 → 용량 산정 → 스프린트 목표 → 후보 범위 선별(우선순위·의존성) → 초과분 분리.
   `templates/스프린트계획.md` 구조로 정리. 용량/속도에 **근거·가정**을 단다.
4. **검수:** `pm-reviewer`로 실행가능성·정합 검수. 🔴 있으면 수정 후 재검수.
5. 편성·할당 쓰기가 필요하면 변경 계획 표로 묶어 승인받고 `/jira-apply`로 반영한다.

## 주의
- **계획만 — 자동 편성 금지.** 스프린트 편성/이슈 할당 쓰기는 승인 게이트(`/jira-apply`)를 거친다.
- 데이터 없는 임의 추정 금지. 가정·근거를 명시하고 불확실은 `(확인 필요)`.

인자(`$ARGUMENTS`)가 있으면 보드/기간/인원으로 해석하고, 부족하면 먼저 묻는다.
