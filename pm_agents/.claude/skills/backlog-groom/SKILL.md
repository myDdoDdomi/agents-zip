# /backlog-groom — 백로그 정리(그루밍) 제안

`backlog-groomer` 에이전트가 백로그를 읽어 중복·우선순위·세분화·정보 누락을 진단하고 개선안을 제안합니다.

## 절차
1. **대상 확인:** 프로젝트키/필터(예: 미배정 백로그)를 확인한다. 모호하면 1~2개만 질문.
2. `backlog-groomer`에게 위임: JQL로 백로그 조회 → 중복·우선순위·세분화·누락 4축을 `현재 → 제안 + 근거` 표로.
3. `templates/백로그정리.md` 구조로 정리. 쓰기로 이어질 제안은 "쓰기 후보" 섹션에 모은다.
4. 핵심 요약을 보고하고 다음 단계(`/pm-review` → 승인 시 `/jira-apply`)를 안내한다.

## 주의
- **읽기 전용 — 제안만.** 자동 적용을 전제하지 않는다. 실제 변경은 승인 게이트(`/jira-apply`)를 거친다.
- 각 제안에 **대상 이슈키 + 현재 값 + 근거**를 단다. 불확실은 `(확인 필요)`.

인자(`$ARGUMENTS`)가 있으면 프로젝트키/필터로 해석하고, 부족하면 먼저 묻는다.
