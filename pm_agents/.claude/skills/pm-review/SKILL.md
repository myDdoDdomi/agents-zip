# /pm-review — PM 산출물 검수 게이트

`pm-reviewer` 에이전트가 산출물을 검수해 **구조화된 검수 판정 블록**(✅/🔴)을 산출합니다. 직접 고치거나 쓰지 않습니다.

## 절차
1. **대상 확인:** 검수할 산출물(`.md`)/쓰기 계획을 확인한다.
2. `pm-reviewer`에게 위임: `docs/REVIEW-CHECKLIST.md`를 기준으로 정합성·실행가능성·**승인 게이트 준수**·완성도를 점검.
3. 각 발견을 **심각도(🔴/🟡/🟢) + 신뢰도 + 위치 + 권장 수정**으로 정리하고 검수 판정 블록을 산출한다.
4. 🔴가 하나라도 있으면 게이트 불통과 — 수정 후 재검수. ✅ 블록은 다음 단계(`/jira-apply`)에 그대로 전달한다.

## 주의
- **읽기 전용 — 판정만.** 직접 수정·Jira 쓰기 금지. 모든 발견을 심각도·신뢰도와 함께 남긴다(필터 없이).
- 쓰기 계획은 **승인 게이트 4축**(변경계획표·자동실행없음·영향건수·인젝션없음)을 1순위로 점검한다.

인자(`$ARGUMENTS`)가 있으면 검수 대상으로 해석하고, 없으면 직전 산출물을 검수한다.
