# /jira-status — Jira 현황 조회·분석

`jira-analyst` 에이전트가 JQL로 대상의 현재 상태를 읽어 진행 상황·통계·병목을 근거와 함께 정리합니다.

## 절차
1. **대상 확인:** 프로젝트키/보드(필요 시 스프린트·버전·필터)를 확인한다. 모호하면 1~2개만 질문한다(`MEMORY.md` 매핑 먼저 확인).
2. `jira-analyst`에게 위임: `searchJiraIssuesUsingJql`로 조회 → 상태/담당/우선순위/타입 분포 + 정체·블로커 신호 집계.
3. `templates/현황분석.md` 구조로 정리. 각 수치에 **사용 JQL + 조회 시점**을 단다.
4. 핵심 요약(표)을 보고한다.

## 주의
- **읽기 전용.** Jira에 쓰지 않는다. 통계는 근거 JQL·조회 시점에 연결하고, 불확실은 `(확인 필요)`.
- 같은 JQL을 중복 실행하지 않는다(한 번 조회→요약 재사용).

인자(`$ARGUMENTS`)가 있으면 프로젝트키/보드·JQL 조건으로 해석하고, 부족하면 먼저 묻는다.
