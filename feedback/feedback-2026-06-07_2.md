## 2. 이슈 상세 (현상 -> 영향 -> 권장 업데이트)

### A. 높음 — mock-prod 패리티 미점검 — 다수 런타임 결함이 여러 게이트를 통과

- **현상:** 이번 세션에서 사용자가 prod(VITE_USE_MOCK=false) 환경에서 직접 발견한 결함 목록:
  `generateMission` 미호출, `getExercise`가 Firestore가 아닌 번들 JSON에서 읽음
  (`firebase.ts` 라인 76~77 `import exercisesJson from '../../assets/data/exercises.json'` 확인),
  `createGym` 메서드 누락, `HomeView toggleFollow` mock-only 구현, `createPost`에서 `photoUrl`·
  `authorNickname` 유실, `suggest*` 메서드들 `return []` 스텁, `energyScore` 표시 타이밍 문제.
  각 결함은 mock 환경에서 통과됐으나 prod에서 별도로 발견됐다.
- **영향:** 사용자가 QA 팬아웃 이전에 다수 결함을 하나씩 직접 발견 — 신뢰도 저하 + 수정 루프
  반복 비용 발생. implementer "기능 완료" 판정과 code-reviewer 게이트 이후에도 prod 결함이 남은
  구조적 미점검 패턴이다.
- **권장 업데이트:**
  - `agents/implementer.md`: 기능 완료 self-check 항목에 **"mock에 있는 모든 메서드가
    firebase.ts에도 동일 시그니처로 구현됐는가 (mock-prod 패리티 체크리스트)"** 를 명문화.
    특히 VITE_USE_MOCK=false 환경에서 CF callable 경로, Firestore 경로, stub-vs-real 분기를
