## 2. 이슈 상세

### A. 높음 — 계약 드리프트 4세대 재발: FEN 시점 불일치 + mock 은폐

**현상:** 인게임 자동 quickeval 토글은 ai_service quickeval에 착수 전(fenBefore) FEN을 전송했으나, ai_service는 착수 후(fenAfter) FEN + uci 쌍을 기대했다. 착수 전 FEN으로는 uci 착수칸이 비어 있어 before 복원 로직이 실패하고, 결과적으로 warnLevel이 항상 conservative(none/info)로 폴백했다. medium 강도 필터가 info까지 걸러 배지가 영영 표시되지 않았다. FE 테스트에서 `expect.any(String)`으로 fen 값을 미검증해 은폐됐다.

**계약 드리프트 세대 요약:**
- 1세대(~06-05): casing 불일치(camelCase vs snake_case)
- 2세대(06-05~06-07): 필수 필드 누락(color)
- 3세대(06-07): 부호/방향 불일치(recoveryDelta 해석)
- 4세대(06-07~06-10): FEN 시점 불일치(before vs after)

**영향:** 같은 경계(FE↔backend↔ai_service)에서 4회 연속 재발. 각 세대가 다른 차원(casing→필드→부호→시점)으로 진화해 기존 계약 점검이 다음 차원을 잡지 못하는 구조. 자동 토글 폐기 후 on-demand 코치(`POST /games/{id}/coach`)로 교체해 현상은 해소됐으나, 구조적 예방 규칙이 없으면 5세대가 다른 엔드포인트에서 재발 가능.

**권장 업데이트:**
- `.claude/agents/dev/code-reviewer.md` — 계약 점검 항목에 "FEN 시점(before/after) + uci 쌍 방향 대조"를 명시. 기존 항목(casing·필수필드·부호방향)에 추가해 4차원 체크리스트화.
- `.claude/agents/dev/implementer.md` — 자가 점검에 추가: "FEN을 다른 서비스에 전달할 때 '착수 전 FEN인가 착수 후 FEN인가'를 주석으로 명시하고, 상대 서비스 스펙과 대조했는가?"
- `CLAUDE.md`(완료 기준) — "FE→backend 경계의 FEN 전달은 반드시 시점(before/after)과 함께 주석을 남기고, 대응 테스트에서 실제 fen 값을 검증(expect.any(String) 금지)한다."

**직전 보고서 비교:** 이슈 B(필수 필드 드리프트)로 지적해 code-reviewer 항목 확장을 권장했으나, 이번 주기에 4세대로 재발. 에이전트 업데이트가 이루어지지 않은 채로 사이클이 돌았을 가능성이 있음(확인 필요).

### G. 중간 — OpenAI 크레딧 0: AI 코치 핵심 기능 비가동 상태

**현상:** OpenAI API 크레딧이 소진된 상태로, `POST /games/{id}/coach`는 source=rule(규칙 폴백)로만 동작 중이다. AI 코치(source=ai, 성향 기반 개인화 설명)가 핵심 신기능이므로 크레딧 미충전 상태에서 실질적 검증이 불가능하다.

**영향:** 기능 구현과 테스트는 완료됐으나 실제 AI 코칭 경험 검증이 프로덕션에서 이루어지지 않고 있다. 규칙 폴백이 사용자에게 노출 중.

**권장 업데이트:**
- `MEMORY.md` §5(미해결) — 현재 항목에 "AI 코치 비가동 상태, 크레딧 충전 후 source=ai 동작 검증 필요"를 추가.
- `.claude/agents/dev/dev-lead.md` — 외부 API 의존 기능 계획에 추가: "외부 API(OpenAI 등) 크레딧/쿼터 상태를 배포 전 확인하고, 크레딧 부족 시 폴백 경험이 허용 수준인지 사전 합의한다."
