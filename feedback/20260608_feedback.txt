IA(정보구조도) Depth 규칙

1. 용어 정의

1Depth: 서비스의 최상위 기능 모듈 단위다. "인증", "온보딩", "홈", "마이페이지"처럼 서비스를 구성하는 대분류 기능 영역이 여기에 해당한다. 1Depth는 화면이 아니라 기능 묶음의 이름이다.

2Depth: 1Depth 모듈 안에서 사용자가 거치는 각 단계(Step)에 해당하는 화면이다. Step끼리는 순서가 있더라도 상하 관계가 아니라 병렬 관계이므로 전부 같은 2Depth에 위치한다.

3Depth: 특정 2Depth 화면 안에서 열리는 하위 화면이다. 해당 Step 내부에서만 접근 가능한 부가 기능이나 옵션 선택 화면이 여기에 해당한다.

2. Step과 Depth의 구분

Step은 하나의 기능을 완료하기 위해 사용자가 순차적으로 거치는 단계다. Step은 순서일 뿐 계층이 아니다. 따라서 Step이 아무리 많아도 같은 Depth에 나란히 놓인다.

Depth가 내려가는 경우는 특정 Step 화면 내부에서 별도의 하위 화면이 열릴 때뿐이다.

예시: 배달의민족 주문하기 기능의 경우 주문메뉴 확인, 배송지 정보 확인, 결제, 주문완료는 순서대로 진행되지만 전부 2Depth다. 배송지 정보 확인 화면 안에서 열리는 주소 검색, 할인쿠폰은 해당 Step의 하위 기능이므로 3Depth다.

3. Type 분류 기준

Page: 앱 화면 자체로 제공되는 독립된 바닥 페이지다. 화면 전환이 일어나며 새로운 화면이 로드된다.

Popup: 사용자가 버튼을 누르거나 특정 옵션을 선택했을 때 현재 화면 위에 오버레이로 열리는 화면이다. 모달, 바텀시트, 토스트 등이 포함된다.

4. 적용 규칙

규칙 1: 1Depth에는 기능 모듈명만 적는다. 화면명을 1Depth에 직접 쓰지 않는다.

규칙 2: 하나의 모듈 안에서 순차적으로 진행되는 화면들은 전부 2Depth로 배치한다. "A 다음에 B가 나온다"는 이유만으로 B를 A의 하위 Depth로 내리지 않는다.

규칙 3: Depth를 내리는 유일한 조건은 "특정 화면 내부에서만 접근 가능한 하위 화면이 존재할 때"다.

규칙 4: Popup 타입의 화면은 자신을 호출한 화면보다 한 단계 아래 Depth에 위치시킨다.

규칙 5: 같은 2Depth 화면에서 열리는 Popup이 여러 개 있으면 전부 같은 3Depth에 나란히 놓는다. Popup끼리 상하 관계를 만들지 않는다.

5. 검증 체크리스트

IA를 수정한 뒤 다음을 확인한다.

첫째, 1Depth에 화면명이 직접 들어가 있지 않은가. 둘째, 순차 Step을 Depth 증가로 처리한 곳이 없는가. 셋째, 모든 화면에 Type(Page 또는 Popup)이 지정되어 있는가. 넷째, Popup이 호출 화면과 같은 Depth에 놓여 있지 않은가. 다섯째, 화면본수를 셀 때 1Depth 모듈명을 화면으로 카운트하고 있지 않은가.

6. 참고 예시

배달의민족 주문하기 기능 IA 구조는 다음과 같다.

1Depth 주문하기 아래에 2Depth로 주문메뉴 확인(Page), 배송지 정보 확인(Page), 결제(Page), 주문완료(Page)가 위치한다. 배송지 정보 확인의 3Depth로 주소 검색(Popup), 할인쿠폰(Popup)이 위치한다.
