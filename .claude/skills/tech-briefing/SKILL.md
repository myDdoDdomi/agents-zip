# /tech-briefing — IT 뉴스 종합 브리핑 발행

사람이 읽는 **IT 뉴스 종합 브리핑**을 발행하는 본부 유틸리티 스킬이다. 한국·해외 IT 전반의
주요 소식을 분야별로 종합하고, **"알아두면 좋은 신개념·용어" 해설**을 곁들여 가독성 있게 펴낸다.
발행 본체는 `tech-news-reporter` 에이전트다.

> **`/update-agents`와 목적이 다르다.** `/update-agents`는 트렌드를 필터·매핑해 **팀 파일을 디벨롭(적용)**한다.
> 이 스킬은 **사람이 읽는 정보 제공 전용** — 팀 파일을 수정하지 않는다.
> **빌트인 웹검색만 쓴다(외부 MCP 불요).** 수집은 `tech-news-reporter`의 `WebSearch`/`WebFetch`로 한다.

## 0. 상태 확인 (diff 기준점)

1. `briefings/`를 `Glob`(`briefing-*.md`)으로 본다.
   - 인자가 `diff`면 → 파일명 **날짜 최댓값**이 **diff 기준점**이다. 그 날짜 이후 새 소식만 수집한다.
   - 인자가 없으면(또는 특정 주제면) → **종합 모드**(현 시점 전체 지형). 주제 인자는 그 주제로 좁힌다.
   - `briefings/`가 비어 있거나 인자 `diff`인데 기존 브리핑이 없으면 → 종합 모드로 폴백한다.

## 1. 발행 — tech-news-reporter

2. `tech-news-reporter`에게 위임한다.
   - **모드**(종합 / diff + 기준 날짜)와 **선택 주제**(인자가 특정 주제면)를 전달한다.
   - 산출: 분야별 종합 + 신개념·용어 해설 브리핑(각 소식에 출처 URL·날짜·"왜 중요" 한 줄).

## 2. 저장 — briefings/

3. 발행물을 `briefings/briefing-YYYY-MM-DD.md`로 저장한다(**오늘 날짜**, 세션 컨텍스트 기준).
   - **같은 날 재실행이면** `briefing-YYYY-MM-DD-2.md`(`-3`…)로 접미해 직전 발행물을 덮어쓰지 않는다(재실행 안전).
   - 이 파일의 날짜가 **다음 diff 기준점**이 된다.

## 3. 화면 요약 표시

4. 사용자 화면에 요약을 보여준다:
   - **분야별 핵심**(분야당 1~2줄) + **🧠 신개념·용어 해설** + **가장 주목할 N개**.
   - 전체 본문이 아닌 핵심만 — 본문은 저장 파일에 있다.

## 4. 보고

5. 저장 경로(`briefings/briefing-YYYY-MM-DD.md`)와 **다음 diff 기준 날짜**를 보고한다.

---

## 인자 (Arguments)

- **없음** → 종합 모드(현 시점 전체 지형).
- **`diff`** → 직전 브리핑 날짜 이후 새 소식만.
- **그 외 텍스트** → 그 주제 집중(예: `/tech-briefing 보안` → 보안 중심 브리핑).

## 산출 형식 — briefings/briefing-YYYY-MM-DD.md

```markdown
# IT 뉴스 브리핑 — YYYY-MM-DD
> 모드: 종합 / diff(기준 YYYY-MM-DD) | 범위: 한국+해외 IT 전반 | 발행: tech-news-reporter
> 한계 고지: 가용 웹 신호 기반 best-effort. 미확인은 (확인 필요)로 표기.

## 🔥 이번 브리핑 주목 N개 (한 줄씩)
1. <소식> — <왜 중요 1줄> (출처)

## 분야별 종합
### AI·에이전트 / Claude·MCP / 언어·런타임 / 프레임워크 / 개발도구 / 보안 / 데이터·인프라 / 디자인 / 마케팅 / 규제·플랫폼
- <소식> — <왜 중요 1줄> (출처 URL, 날짜)

## 🧠 알아두면 좋은 신개념·용어 (Glossary)
- **<용어>**(원어) — <1~3문 해설> (출처)

## 참고·다음 브리핑 메모
- 다음 diff 기준 날짜: YYYY-MM-DD / 이번에 다 못 다룬 주제: …
```

## 주의
- **정보 제공 전용.** 팀 파일·CLAUDE.md를 수정하지 않는다(디벨롭은 `/update-agents`).
- **출처·날짜 필수, 추측 금지.** 미확인은 `(확인 필요)`로 표기한다.
- **분야 균형.** 한 분야에 치우치지 않고 한국·해외를 폭넓게 다룬다.
- **재실행 안전.** 같은 날 재실행은 `-2`(`-3`…) 접미로 직전 발행물을 보존한다.
