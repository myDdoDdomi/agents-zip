# /resume-review — 이력서·포트폴리오 검수 게이트

`resume-reviewer` 에이전트가 초안을 근거·JD와 대조해 4대 항목을 점검하는 **품질 게이트**입니다.

## 절차
1. `docs/REVIEW-CHECKLIST.md`를 기준으로 삼는다.
2. 검수할 초안 + 근거(GitHub 추출·포트폴리오 분석·JD 분석)를 함께 받는다. 근거가 없으면 먼저 요청한다.
3. `resume-reviewer`에게 위임 → 4대 점검:
   - **사실 정합**(근거와 일치, 부풀린 기여도·미확인 수치) / **기업 적합도**(JD 필수·키워드 반영)
   - **과장·허위·금칙·민감정보** / **완성도·일관성**(템플릿·시제·모호어)
4. 심각도(🔴 필수 / 🟡 권장 / 🟢 제안) 부여. **🔴가 하나라도 있으면 게이트 불통과** — 수정 후 재검수.
5. 통과 시 안내: `/drive-sync`로 Drive 반영.

> **거버넌스(핵심):** 근거 없는 경력·성과·과장 기여도를 반드시 잡아 🔴로 처리한다. (→ CLAUDE.md §2, §10)
> 검수자는 직접 고치지 않고 "무엇을 어떻게"만 짚는다(수정은 작성자).

인자(`$ARGUMENTS`)가 있으면 검수 대상으로 사용하고, 없으면 무엇을 검수할지 먼저 묻는다.
