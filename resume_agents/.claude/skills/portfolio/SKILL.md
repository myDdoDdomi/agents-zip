# /portfolio — 기업 맞춤 포트폴리오 문서 작성

`resume-lead`가 근거·정렬 기준을 정리해 `portfolio-writer`에게 포트폴리오 문서를 맡깁니다.

## 절차
1. `templates/포트폴리오.md` 구조를 확인한다.
2. **근거 확인**: GitHub 경력·포트폴리오 분석·JD 분석 결과가 있는지 본다. 없으면 먼저 수집하거나
   핵심 근거가 비면 `(확인 필요)`로 막고 사용자에게 묻는다.
3. `resume-lead`가 근거 요약 + JD 정렬 기준을 만들어 `portfolio-writer`에게 위임한다.
   - JD에 맞는 프로젝트 선별·정렬, 문제→역할→접근/기술→결과(임팩트) 서사. 수치는 **확인값만**, 기여도 사실대로.
   - 각 프로젝트에 근거 출처 메모(검수용)와 링크.
4. 완료 후 안내: `/resume-review` 검수 → `/drive-sync` Drive 반영.

> **거버넌스(절대):** 없는 프로젝트·성과·역할을 만들지 않는다. 근거 없으면 `(확인 필요)`. (→ CLAUDE.md §2)
> 사이트/코드 구현은 `dev_agents`, 비주얼은 `design_agents`로 핸드오프.

인자(`$ARGUMENTS`)가 있으면 지원 회사/직무로 사용하고, 없으면 어느 회사용인지 먼저 묻는다.
