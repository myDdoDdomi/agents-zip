# /drive-sync — Docs 서식·Drive 반영

`docs-formatter` 에이전트가 최종본을 **Google Docs 네이티브 서식**으로 정돈해 **Drive에 저장**합니다.

## 절차
1. `docs/DOC-FORMATTING.md` 표준을 절대 기준으로 삼는다.
2. **검수 판정 블록 확인(상태 핸드오프)**: `resume-reviewer`의 **구조화된 검수 판정 블록**
   (`<!-- REVIEW-VERDICT -->` ✅/🔴 + 사유 + 반영 가능 여부)을 본다.
   - **✅ 통과 블록이 이미 전달됐으면 "검수했냐"를 재질문하지 말고 바로 진행**한다(반복 질문 금지).
   - 블록이 없거나 🔴가 남아 있으면 반영하지 않고 돌려보낸다.
3. **반영 대상은 검수 통과한 평문 `.md`.** 작성 단계 산출 `.md`를 단일 원본으로 쓴다.
4. 대상 폴더·파일명(예: `[회사]_[직무]_이력서_v1`)·새 문서/기존 갱신 여부를 확인한다. 모호하면 1~2개만 질문.
5. `docs-formatter`에게 위임 → **공식 양식(.docx→Docs)은 `.md`를 마크다운 import로 새 Doc 생성**을
   기본으로 하고(셀 직접 편집 회피), 커버 헤더·제목 스타일(named style)·네이티브 표·포인트색·여백을 적용한다.
   - 소규모 수정은 `find_and_replace_doc` 등 **부분 편집 우선**, 구조 변경 시에만 전체 재import.
   - 마크다운/ASCII 기호를 본문에 남기지 않는다. 인덱스 안전 절차(tab_id 확인, 텍스트→표 순서, 삽입 후 재inspect)를 따른다.
6. **덮어쓰기 전 확인**: 기존 문서를 갱신할 땐 현재 내용을 읽어 비교하고 손실 위험을 알린 뒤 진행한다.
7. 저장 후 문서 링크와 적용 서식 요약을 보고한다.

> **거버넌스:** 검수 통과본만 반영. 내용을 임의로 바꾸지 않는다(서식·반영만). (→ CLAUDE.md §2, §8)
> MCP 미연결이면 서식 적용된 마크다운을 산출하고 사용자가 복사해 붙여넣도록 안내한다(폴백).

인자(`$ARGUMENTS`)가 있으면 반영 대상·폴더로 사용하고, 없으면 무엇을 어디에 올릴지 먼저 묻는다.
