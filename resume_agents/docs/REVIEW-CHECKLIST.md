# 이력서·포트폴리오 검수 체크리스트

> `resume-reviewer` 에이전트가 **절대 기준**으로 삼는 체크리스트입니다. `/resume-review`가 이를 호출합니다.
> 검수자는 직접 고치지 않고 "무엇을 어떻게 고칠지"를 심각도와 함께 짚습니다(수정은 작성자).

## 심각도
- 🔴 **필수**: 사실 불일치·근거 없음·과장·허위·민감정보 노출. **하나라도 있으면 게이트 불통과.**
- 🟡 **권장**: 기업 적합도 미흡·완성도/일관성 문제.
- 🟢 **제안**: 표현·가독성 개선.

---

## 1. 사실 정합 (🔴 핵심 — 이 팀 1순위)
- [ ] 모든 경력·성과·스킬이 **근거(레포/포트폴리오/사용자 확인)와 일치**하는가
- [ ] 근거 없는 항목이 있는가 → 있으면 🔴 (삭제 또는 `(확인 필요)` 처리 요구)
- [ ] 기여도(주도/공동/포크)가 사실대로인가 → 부풀림은 🔴
- [ ] 수치·성과가 **확인값**인가 → 임의 수치는 🔴
- [ ] 토이/연습 프로젝트를 상용/대규모로 오해시키지 않는가

## 2. 기업 적합도 (🟡)
- [ ] JD의 **필수요건**이 이력서/포트폴리오에 반영됐는가
- [ ] JD **핵심 키워드**가 적절히 정렬·강조됐는가
- [ ] 직무와 무관한 내용이 앞에 배치돼 있지 않은가
- [ ] 회사 인재상·도메인과 연결돼 있는가

## 3. 과장·허위·금칙·개인정보 (🔴/🟡)
- [ ] 과장 형용사("최고의","압도적") 남발이 없는가
- [ ] 미검증 효능·근거 없는 단정("업계 1위")이 없는가 → 있으면 🔴
- [ ] 민감 개인정보(주민번호·상세주소·가족사항 등)가 불필요하게 노출되지 않았는가 → 🔴
- [ ] 부적절·차별적 표현이 없는가

## 4. 완성도·일관성 (🟡/🟢)
- [ ] 템플릿 필수 섹션이 채워졌거나 `(TBD)`/`(확인 필요)`로 명시됐는가
- [ ] 용어·시제가 일관되는가(과거 경력=과거형)
- [ ] 모호어("다양한","등","적절히")가 구체화됐는가
- [ ] 링크·증빙이 달렸고 접근 가능 여부가 표시됐는가

---

## 판정
- **🔴 0건** → **통과** (Drive 반영 가능, `/drive-sync`)
- **🔴 1건 이상** → **불통과** (작성자 수정 후 재검수)

> 남은 `(확인 필요)`는 **검증 책임자(보통 사용자 본인)** 를 명시해 사용자에게 확인을 요청합니다.
