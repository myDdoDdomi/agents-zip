# GitHub MCP 연결 가이드 (경력 근거 읽기 전용)

`github-career-extractor`가 GitHub 레포·기여·스택을 **경력 근거**로 읽으려면 **GitHub 공식 원격
MCP 서버**를 연결합니다. 팀 공유 설정은 루트 `.mcp.json`에 들어 있고, **최초 1회 브라우저 OAuth**
(개인 액세스 토큰 PAT 불필요)로 인증합니다.

> 사용하는 서버: **GitHub 공식 원격 MCP** — `https://api.githubcopilot.com/mcp/` (HTTP 타입, OAuth).
> 이 팀은 레포·기여를 **읽기만** 합니다. 이슈/PR/머지 등 쓰기 작업은 하지 않습니다.

---

## A. Claude Code에서 승인 & 인증
1. 이 폴더에서 `claude` 실행 → `.mcp.json`의 `github` 서버 사용을 **승인(Yes)**.
2. `/mcp`로 서버 상태를 확인하고, GitHub **OAuth(브라우저)** 로그인/동의를 진행합니다.
   - PAT를 만들 필요가 없습니다(원격 MCP가 OAuth로 처리).
3. 인증 후 `/mcp`에서 `connected` 확인.

## B. 권한 최소화 권장
- 이 팀은 **읽기 전용**으로 충분합니다. OAuth 동의 시 과한 쓰기 권한을 부여하지 마세요.
- 공개 레포만 다룬다면 비공개 레포 접근 권한은 굳이 부여하지 않아도 됩니다(필요할 때만).
- 가능하면 **개인 일반 계정**으로 인증하고, 작업이 끝나면 토큰/세션을 정리합니다.

## C. 연결 확인
```
/github-career  <본인 GitHub ID>
```
레포 목록·언어·기여가 보이면 성공입니다.

---

## D. 문제해결
| 증상 | 원인 | 해결 |
|---|---|---|
| `/mcp`에 서버 안 보임 | 승인 안 함 | `claude` 재시작 후 승인 |
| OAuth 창이 안 뜸 | 네트워크/방화벽 | 브라우저 수동 인증 안내 따르기, 재시도 |
| 비공개 레포 안 보임 | 권한 미부여 | 비공개 레포가 필요하면 OAuth 동의에서 해당 권한 허용 |
| 403/429 | 레이트리밋 | 잠시 후 재시도, 조회 범위 축소 |

## E. 폴백 (MCP 없이)
MCP를 못 쓰면, 사용자가 **레포 목록·README·기여 내용을 붙여넣으면** `github-career-extractor`가 그
입력을 근거로 경력 근거를 정리합니다(추측 없이 붙여넣은 사실만 사용).

---

## F. 거버넌스 (이 팀 1순위와 직결)
- **실제 레포·커밋·기여만 근거**로 씁니다. fork·튜토리얼·연습 레포를 "주도 개발"로 부풀리지 않습니다.
- 공동 레포는 **본인 기여 기준**으로 역할을 기술합니다. 불확실하면 `(확인 필요)`.
- 쓰기 작업(이슈/PR/머지)은 하지 않습니다 — 이 MCP는 근거 읽기 용도입니다.

> ⚠️ 토큰/세션 정보는 절대 레포에 커밋하지 마세요. `.gitignore`로 차단돼 있습니다.
