# aiot-agents 📡🤖

AIoT(AI + IoT) 제품을 **기획부터 시스템 설계, 엣지AI, 펌웨어·연결성·클라우드 설계·참조 구현, 디바이스 보안까지** 한 팀으로 도와주는 **AI 엔지니어링 팀**입니다. 산업IoT·스마트홈·웨어러블·로보틱스 등 **전 분야(범용)** 를 다룹니다.

## 🟢 Claude Code 처음이세요? — 설치 1분

터미널을 열고 **자기 컴퓨터에 맞는 아래 한 줄을 복사 → 붙여넣기 → Enter** 하세요.

## 🚀 이 팀 쓰는 법 (3단계)

1. **이 폴더를 `claude`로 엽니다.** 이 `aiot_agents` 폴더에서 터미널을 열고 `claude`를 입력하세요. 팀원(에이전트)과 단축 명령(스킬)이 자동으로 준비됩니다.
2. **(필요할 때만) 문서 조회 MCP를 씁니다.** SoC·RTOS·클라우드 SDK·프로토콜 최신 문서를 읽어오는 **Context7**이 연결돼 있어요(읽기 전용). 키 없이도 동작하며, 더 많이 조회하려면 환경변수로 API 키를 넣을 수 있습니다.
3. **한국어로 시킵니다.** 예를 들어 이렇게요.
   ```
   배터리로 동작하는 스마트홈 센서 시스템 아키텍처 잡아줘
   ```
   → (요구·제약 정리) → 시스템 아키텍처 → 프로토콜/엣지AI/펌웨어/클라우드 설계 → 보안 위협모델 → 검토 순으로 알아서 진행합니다.

**결론: 외울 건 없습니다. 그냥 한국어로 하고 싶은 걸 말하세요.**

## ⚖️ 우리 범위 vs 다른 팀에 넘기는 일 (핸드오프)

이 팀은 **설계·참조 구현**이 핵심입니다. 아래는 다른 팀으로 넘깁니다.

| 넘기는 일 | 받는 팀 |
|---|---|
| **양산 펌웨어/클라우드 구현·정식 코드리뷰·리팩터·CI** | `dev_agents` (복사해서 쓰는 팀) |
| **정식 침투 테스트(펜테스트)·레드/블루팀** | `dev_agents` (`/pentest`) |
| **정식 기능/기술 명세서·Drive 반영·일정계획** | `docs_agents` (직접 여는 팀) |
| 디바이스 UI/대시보드 **비주얼·디자인** | `design_agents` |
| 펌웨어/디바이스 **QA·테스트 자동화** | `QA_agents` (복사) |

> `firmware-embedded-engineer`·`iot-cloud-data-engineer`는 **참조 구현·프로토타입까지** 만듭니다. 양산 코드베이스 전반·정식 검증으로 넘어가는 시점은 그룹장이 알려 위 팀으로 핸드오프/조립합니다.

## 🤝 다른 팀과 함께 쓰기 (멀티팀 조립)

이 AIoT팀을 **개발(dev)·QA·문서(docs) 등 다른 팀과 한 프로젝트에서 같이** 쓰고 싶을 때 읽으세요.
(AIoT팀만 혼자 쓸 거면 — 이 `aiot_agents` 폴더를 그냥 직접 열면 됩니다. 조립은 필요 없어요.)

AIoT팀은 **폴더를 직접 여는 팀(모델 A)** 이라, 다른 팀과 합칠 땐 **에이전트(팀원)만 옮기면 부족**해요.
슬래시 명령(스킬)·설계 템플릿·MCP 설정까지 함께 옮겨야 제대로 동작합니다.

### 무엇을 복사하나 (4가지)

대상 프로젝트(여러 팀을 같이 쓸 폴더)로 아래 4가지를 옮깁니다.

1. **① 팀원(에이전트):** `aiot_agents/.claude/agents/*` → `대상-프로젝트/.claude/agents/aiot/`
2. **② 단축 명령(스킬):** `aiot_agents/.claude/skills/*` → `대상-프로젝트/.claude/skills/` (슬래시 명령이 동작하려면 필수)
3. **③ 설계 템플릿:** `aiot_agents/templates/*` → `대상-프로젝트/templates/` (에이전트가 참조하는 상대경로 유지)
4. **④ MCP 설정:** `aiot_agents/.mcp.json` → `대상-프로젝트/.mcp.json`에 **합칩니다(병합)**.
   비밀값은 들어 있지 않고 `${ENV}` 형태로 두며, 같은 키가 이미 있으면 사용자에게 확인합니다.
   + 본부의 `team-assemble` 스킬 폴더도 대상 `.claude/skills/`로 함께 복사하세요.

> `dev_agents`·`QA_agents`처럼 **복사해서 쓰는 팀(모델 B)** 은 `agents/*.md`만 옮기면 됩니다(스킬·템플릿·MCP 없음).

### 실행

대상 프로젝트에서 **`/team-assemble`를 한 번** 실행하면 됩니다.
→ 팀원 이름이 겹쳐도 자동으로 정리되고(충돌 해결), 협업용 `CLAUDE.md`·`MEMORY.md`가 자동으로 만들어집니다.
팀을 더하거나 빼면 **다시 실행만** 하면 됩니다(여러 번 돌려도 안전).

### 알아둘 점

- AIoT팀의 `CLAUDE.md`·`MEMORY.md`는 따라가지 않습니다 — 협업용 `CLAUDE.md`의 자동 블록과 공통 규칙으로 대체돼요.
- 회고 담당(`feedback-reporter`)은 조립 시 **하나로 통합**됩니다. `/feedback-agents` 한 번이면 합쳐진 **모든 팀**을 회고해 본부로 보고됩니다. (스킬을 안 옮겼어도 `/team-assemble`이 통합 `/feedback-agents`를 자동 생성합니다.)

### 📡 MCP 주의 — Context7는 재인증 거의 불필요

AIoT팀의 `.mcp.json`에는 **Context7(읽기 전용 문서 조회) MCP** 설정이 들어 있어요. 키 없이도 동작하므로 **추가 인증이 거의 필요 없습니다**(레이트리밋 상향 시에만 `CONTEXT7_API_KEY` 환경변수). GitHub 레퍼런스 레포를 읽으려면(선택) `${GITHUB_PAT}`만 환경변수로 넣으면 됩니다(읽기 전용 권장). **비밀값은 레포에 안 들어가요.**

> 더 자세히는 본부의 루트 `README.md`/`CLAUDE.md`(§2-C 조립)와 `/team-assemble` 스킬을 보세요.

## 🛠 고급(개발자용)

### 🧠 팀 메모리 (MEMORY.md) 사용법

이 팀은 두 층의 메모리를 쓴다.
- **CLAUDE.md** — 사람이 쓰는 안정적 규칙·라우팅(매 세션 전체 로드).
- **MEMORY.md** — 작업하며 쌓이는 학습(검증된 함정·우회법·환경 매핑·반복 교정). CLAUDE.md가 `@./MEMORY.md`로 자동 로드한다.

| 항목 | 내용 |
|---|---|
| **언제 적나** | 같은 실수 2회 · 환경/도구 특이사항(SoC 빌드/플래시 함정·양자화 손실 등) · 매 세션 다시 설명하게 되는 것 · 검수가 잡아준 것 |
| **어떻게** | 해당 섹션에 **한 줄**(가능하면 날짜·근거). 규칙으로 굳으면 CLAUDE.md로 승격 |
| **확인** | 세션 중 `/memory`로 로드된 메모리 파일 확인 |
| **용량** | 200줄 이하 유지. 초과 시 `memory/<주제>.md`로 분리(필요할 때만 읽힘) |
| **개인 메모리** | 개인·머신 한정 학습은 Claude Code 자동 메모리(`~/.claude/projects/.../memory/`)가 별도 관리(레포 비커밋). 팀 공유 학습은 MEMORY.md |
