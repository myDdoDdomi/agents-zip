# /edge-ai-plan — 엣지AI 배치 계획

`edge-ai-engineer` 에이전트를 사용해 **온디바이스/엣지 AI 배치 계획**을 만듭니다.

## 절차
1. `templates/엣지AI배치계획.md` 구조를 확인한다.
2. 태스크·정확도 목표·타깃 하드웨어(MCU/NPU/DSP)·전력/레이턴시 제약을 확인한다(없으면 질문).
3. `edge-ai-engineer`에게 위임해 **엣지 vs 클라우드 배치 결정**·모델 선택·경량화(양자화/프루닝/증류)·
   하드웨어 적합도(RAM/Flash/MAC 예산)·추론 예산·정확도 검증 계획을 정의한다.
4. 메모리/전력 영향은 `firmware-embedded-engineer`, 데이터/재학습 루프는 `iot-cloud-data-engineer`와 협의.

## 주의
- 양자화에 따른 **정확도 손실 측정·허용 기준**을 반드시 정의(임의 단정 금지).
- 대규모 학습 파이프라인 운영은 범위 밖 — 필요 시 핸드오프를 표시한다.

인자(`$ARGUMENTS`)가 있으면 대상으로 사용하고, 없으면 먼저 묻는다.
