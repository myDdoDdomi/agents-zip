# /design-research — 디자인 레퍼런스·인사이트 리서치

`design-researcher` 에이전트를 사용해 **디자인 레퍼런스·트렌드·인사이트**를 조사하고 우리 서비스에 맞춘 적용 제안을 정리합니다.

## 절차
1. `templates/디자인리서치.md` 구조를 확인한다.
2. 리서치 목표·우리 서비스 맥락(타깃·브랜드·제약)을 모은다. (있으면) 기존 디자인 시스템 요약,
   기존 Figma 화면은 `figma-operator` 추출 요약을 참조한다.
3. `design-researcher`에게 위임해 소스를 세 종류로 나눠 **출처 URL과 함께** 수집한다:
   ① 구현/컴포넌트 레지스트리(21st.dev·shadcn 등 실제 코드 있음) ② 라이브 제품 UI(Mobbin 등) ③ 영감·비주얼(Dribbble 등).
   **패턴→원리→시사점**으로 인사이트를 도출한 뒤 **적용 제안(차용/회피·트레이드오프·MoSCoW)** 을 정리한다.
   정보 부족 시 핵심부터 사용자에게 질문한다.
4. **경로 판정(design-lead):**
   - **(A) 목업-우선** → 완료 후 안내: `/ideate`(컨셉화) → `/design-tokens`·`/design-system`(정의).
   - **(B) 구현 레퍼런스 → dev 직행** → 실제 구현 컴포넌트의 **라이선스 확인** 후 템플릿 **§개발 핸드오프 노트**를 채워
     `dev_agents` 팀으로 인계(목업 단계 스킵 가능). 멀티팀이면 본부 `/team-assemble`로 design+dev 조립.

인자(`$ARGUMENTS`)가 있으면 리서치 목표/대상으로 사용하고, 없으면 무엇을 조사할지 먼저 묻는다.
