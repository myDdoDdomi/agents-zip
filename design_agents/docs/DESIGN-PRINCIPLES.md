# 디자인 원칙 (팀 공통)

모든 디자인 에이전트가 따르는 기준입니다. 그룹장(`CLAUDE.md`)의 운영 원칙과 함께 적용합니다.

## 1. 기존 시스템 재사용 우선
- 새 컴포넌트·토큰을 만들기 전에 **기존 Figma 변수·코드 컴포넌트를 먼저 읽는다.**
- 보고서 결론: 디자인-개발의 최대 병목은 **drift**다. "새로 만들기"보다 "재사용·연결"이 기본값.

## 2. 토큰 계층
- 원시(primitive) → 의미(semantic) → 컴포넌트(component) 3계층으로 토큰을 정의한다.
- 의미 토큰을 통해 테마/브랜드 변경이 한 곳에서 전파되게 한다.

## 3. 접근성 내장 (WCAG 2.2)
- 본문 대비 ≥ 4.5:1, 큰 글자·UI ≥ 3:1. 색에만 의존하지 않는 정보 전달.
- 키보드 도달·포커스 표시, 터치타겟 ≥ 24px(권장 44px).
- 모든 화면은 빈/로딩/에러/성공 **상태**를 설계한다.

## 4. 디자인-코드 정합 (Code Connect)
- 디자인 시스템 컴포넌트는 실제 코드 컴포넌트와 매핑한다(prop/variant/토큰).
- 매핑 누락·불일치(drift)를 주기적으로 점검한다.

## 5. 외부 영향은 게이트
- Figma 쓰기·토큰 push·코드 매핑 커밋·덮어쓰기는 **사람 승인** 후 실행한다.
- 되돌리기 어려운 작업일수록 변경 내용·영향 범위를 먼저 보여준다.

## 6. 투명성
- 추측 금지(`(확인 필요)` 표기), 미검증·가정 명시, 측정 가능한 기준으로 판정.
