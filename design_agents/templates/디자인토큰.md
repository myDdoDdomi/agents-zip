## 4-1. Tailwind 정합 (Figma Variable ↔ Tailwind 토큰 1:1)
> 코드가 Tailwind를 쓰면, 프로젝트의 Tailwind 버전을 먼저 확인해 점검한다.
> - **(v3 이하)** 시맨틱 토큰 이름이 `tailwind.config` `theme.extend` 키와 1:1 대응되는지 점검한다.
> - **(v4 채택 시)** JS config 폐기 — CSS `@theme` 지시어의 CSS 변수(예: `--color-action-primary`)와 1:1 대응되는지 점검한다.

| Figma Variable | Tailwind 토큰 키 (v3: extend.키 / v4: --var명) | 일치 여부 |
|---|---|---|
| `color/action/primary` | v3 `extend.colors.action.primary` / v4 `--color-action-primary` | ✅/❌ |
| `radius/md` | v3 `extend.borderRadius.md` / v4 `--radius-md` | ✅/❌ |

> 다크모드: Tailwind 다크 전략(`class`/`media` + CSS 변수)에 모드별 값이 어떻게 노출되는지 비고에 적는다. `(확인 필요: 실제 config 키는 소스 기준)`
